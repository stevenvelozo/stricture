# Command: TestObjectContainers

Generate per-table JSON files containing test fixture objects.

**Source:** `Stricture-Generate-TestObjectContainers.js`

## Usage

```bash
node Stricture.js -i MeadowModel-Extended.json -c TestObjectContainers -f ./output/ -o TestFixtures
```

## Input

A JSON model file (basic or extended).

## Output

One JSON file per table: `{OutputFileName}{TableName}.json`

Each file contains an array of 25 test objects.

## Generated Test Objects

Each object has every column from the table populated with a type-appropriate
default value:

| MicroDDL Type | Default Value       | Notes                         |
|---------------|---------------------|-------------------------------|
| ID            | Sequential (0-24)   | Auto-incremented per object   |
| GUID          | Generated UUID      | Unique per object via Fable   |
| ForeignKey    | `0`                 |                               |
| Numeric       | `0`                 |                               |
| Decimal       | `0.0`               |                               |
| String        | `""`                |                               |
| Text          | `""`                |                               |
| DateTime      | `null`              |                               |
| Boolean       | `false`             |                               |

## Notes

- The primary key column is auto-incremented from 0 to 24 across the 25
  objects.
- GUID columns receive a unique UUID for each object, generated by the
  Fable UUID service.
- All other columns use zero/empty/null defaults. These fixtures provide
  structurally valid objects for unit tests but do not contain meaningful
  test data.
- The 25-object count is hardcoded.
